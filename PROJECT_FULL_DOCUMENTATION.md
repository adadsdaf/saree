## نظرة عامة

**واصل** هو منصة توصيل طعام متكاملة تدعم ثلاثة أطراف:
- **العميل**: يتصفح المطاعم ويطلب الطعام ويتابع توصيله
- **السائق**: يستقبل الطلبات وينجزها ويدير أرباحه
- **المدير**: يتحكم في المنصة بالكامل من مطاعم وسائقين وطلبات وتقارير

**المكدس التقني:**
| الطبقة | التقنية |
|--------|---------|
| الواجهة الأمامية | React 18 + Vite + Tailwind CSS + Shadcn UI |
| الواجهة الخلفية | Node.js + Express.js + TypeScript |
| قاعدة البيانات | PostgreSQL (Neon) + Drizzle ORM |
| التوثيق | Passport.js (Local Strategy) |
| الوقت الفعلي | WebSockets (ws) |
| الجلسات | express-session + connect-pg-simple |

## تطبيق العميل

### 🏠 الصفحة الرئيسية (Home)
**ما يراه العميل:**
- قسم ترحيبي مع شريط بحث
- شبكة تصنيفات (برجر / بيتزا / مشاوي / حلويات...)
- قائمة المطاعم المتاحة مع:
  - صورة المطعم + اسمه + تقييمه
  - وقت التوصيل المتوقع
  - رسوم التوصيل
  - الحد الأدنى للطلب
  - حالة المطعم (مفتوح / مغلق)
- العروض والخصومات الخاصة النشطة

---

### 🔍 البحث (Search)
- البحث بالاسم عن مطاعم أو منتجات
- تصفية النتائج بالتصنيف
- عرض فوري للنتائج

---

### 🍕 صفحة المطعم (Restaurant)
**ما يراه العميل:**
- معلومات المطعم الكاملة (صورة / وصف / ساعات العمل)
- قائمة الطعام مقسمة حسب الأقسام
- لكل منتج: الاسم + الصورة + السعر + وصف + خيارات الأحجام
- زر إضافة للسلة مع إمكانية تعديل الكمية
- تقييمات العملاء السابقين

---

### 🛒 السلة والطلب (Cart & Checkout)
**وظائف الصفحة:**
- عرض المنتجات المختارة مع إمكانية تعديل الكمية أو الحذف
- ملخص السعر: مجموع المنتجات + رسوم التوصيل + الخصم + الإجمالي
- اختيار عنوان التوصيل (من المحفوظة أو إدخال جديد)
- إدخال كوبون خصم والتحقق منه فوراً
- اختيار طريقة الدفع (كاش / بطاقة / محفظة)
- إمكانية جدولة الطلب لوقت لاحق
- إضافة ملاحظات خاصة للمطعم أو السائق
- تأكيد الطلب وإرساله

---

### 📍 تتبع الطلب (Order Tracking)
**وظائف الصفحة:**
- عرض حالة الطلب بشكل بصري (تسلسل زمني):
  - ✅ تم استلام الطلب
  - ✅ قيد التجهيز
  - ✅ السائق في الطريق
  - ✅ تم التوصيل
- خريطة تفاعلية تُظهر موقع السائق في الوقت الفعلي
- بيانات السائق (اسم + هاتف + تقييم)
- زر إلغاء الطلب (في المراحل الأولى)
- نافذة تقييم المطعم والسائق بعد التوصيل

---

### 👤 الملف الشخصي (Profile)
- تعديل البيانات الشخصية (الاسم / الهاتف / البريد)
- إدارة العناوين المحفوظة (إضافة / تعديل / حذف / تعيين افتراضي)
- سجل الطلبات السابقة
- رصيد نقاط الولاء
- إعدادات اللغة والمظهر

---

### 🔐 تسجيل الدخول والتسجيل (Auth)
- تسجيل دخول برقم الهاتف
- دعم تسجيل الدخول بـ Google
- دعم تسجيل الدخول بـ Apple
- إنشاء حساب جديد

---

### 🎁 خدمة "واصل لي" (Wasel Li)
> خدمة مخصصة لإرسال أي شيء (ليس بالضرورة طعاماً)
- العميل يحدد نقطة الالتقاط ونقطة التوصيل
- يصف ما يريد إرساله
- السائق يرسل عرض سعر أو المدير يحدد الرسوم
- متابعة الطلب بنفس آلية الطلبات العادية

## لوحة تحكم المدير

### 📊 لوحة الإحصائيات الرئيسية (Dashboard)
**البيانات المعروضة:**
- إجمالي الطلبات اليوم / الأسبوع / الشهر
- إجمالي الإيرادات والعمولات
- عدد العملاء النشطين
- عدد السائقين المتاحين
- أكثر المطاعم مبيعاً
- رسوم بيانية لحركة الطلبات عبر الزمن
- تقارير ربحية قابلة للتصفية حسب الفترة

---

### 🍽️ إدارة المطاعم
**الوظائف:**
- إضافة مطعم جديد مع كامل بياناته (صورة / موقع / ساعات عمل / رسوم)
- تعديل بيانات المطعم
- فتح / إغلاق المطعم يدوياً
- تحديد نسبة العمولة لكل مطعم
- حذف أو تعطيل مطعم
- **إدارة قوائم الطعام:**
  - إضافة / تعديل / حذف المنتجات
  - إدارة الأقسام داخل القائمة
  - تحديد الأسعار والصور والخيارات

---

### 🏷️ إدارة التصنيفات
- إضافة تصنيفات جديدة مع صورة وأيقونة
- تعديل ترتيب عرض التصنيفات
- تفعيل / تعطيل التصنيفات

---

### 📦 إدارة الطلبات
**الوظائف:**
- عرض جميع الطلبات مع التصفية (حسب الحالة / المطعم / التاريخ)
- تغيير حالة أي طلب يدوياً
- تعيين سائق لطلب معين يدوياً
- ضبط الأسعار في حالة النزاعات
- عرض تفاصيل أي طلب كاملة
- إعادة ترقيم الطلبات

---

### 🚗 إدارة السائقين
**الوظائف:**
- إضافة سائق جديد بكامل بياناته
- تعديل بيانات السائق (مركبته / معلومات الاتصال)
- تفعيل / تعطيل حساب السائق
- تحديد نمط دفع السائق (راتب ثابت / عمولة)
- مراقبة مواقع السائقين على خريطة مباشرة
- إدارة المستحقات المالية والسحوبات

---

### 💰 الإدارة المالية
**الوظائف:**
- عرض تقارير الإيرادات والعمولات
- الموافقة على طلبات السحب للسائقين والمطاعم
- إدارة حسابات المطاعم (الأرباح / العمولات / الكشوفات)
- تعديل نسب العمولة لكل طرف
- تقارير مالية مفصلة قابلة للتصدير

---

### 🎟️ إدارة الكوبونات والعروض
**الكوبونات:**
- إنشاء كوبون خصم بكود مخصص
- تحديد نوع الخصم (نسبة مئوية أو مبلغ ثابت)
- تحديد فترة الصلاحية وعدد مرات الاستخدام
- تقييد الكوبون بتصنيف أو مطعم معين
- تفعيل / تعطيل كوبون فوراً

**العروض الخاصة:**
- إنشاء بانرات عروض تظهر في الصفحة الرئيسية
- ربط العرض بمطعم أو منتج أو تصنيف
- جدولة العروض لأوقات محددة

---

### 🚚 إدارة رسوم التوصيل
- إنشاء مناطق توصيل جغرافية
- قواعد التسعير بالمسافة (سعر لكل كيلومتر)
- رسوم ثابتة لمناطق معينة
- إعدادات الحد الأدنى والأقصى للرسوم

---

### 🔔 نظام الإشعارات
- إرسال إشعار جماعي لجميع العملاء
- إرسال إشعار مستهدف لعميل أو سائق معين
- إشعارات تلقائية بتغيير حالة الطلبات
- سجل الإشعارات المرسلة

---

### 🎖️ نظام نقاط الولاء
- إضافة نقاط يدوياً لعميل معين

## تطبيق السائق

### 🏠 لوحة تحكم السائق (Dashboard)
**ما يراه السائق:**
- زر تبديل الحالة: متاح ✅ / غير متاح ❌
- ملخص سريع للأرباح اليومية
- عدد الطلبات المنجزة اليوم
- الطلبات النشطة حالياً
- إشعارات الطلبات الجديدة في الوقت الفعلي (WebSocket)

---

### 📋 الطلبات المتاحة (Available Orders)
- قائمة بالطلبات المعينة له من قبل المدير
- تفاصيل كل طلب: المطعم + العميل + المسافة + الأجرة المتوقعة
- زر قبول الطلب
- الطلبات مُحدَّثة تلقائياً بدون إعادة تحميل (WebSocket)

---

### 🚴 الطلبات النشطة (Active Orders)
**دورة عمل السائق مع الطلب:**
1. **قبول الطلب** — يُعلِم المطعم أن السائق في الطريق
2. **وصول للمطعم** — تحديث الحالة
3. **استلام الطلب** → تغيير الحالة إلى "في الطريق"
4. **التوصيل** → تغيير الحالة إلى "تم التوصيل"

- دعم طلبات "واصل لي" (التوصيل الخاص) بنفس المنطق
- إرسال الموقع الجغرافي تلقائياً أثناء التوصيل

---

### 💼 المحفظة والأرباح (Wallet)
**ما يعرضه التطبيق:**
- الرصيد الحالي
- إجمالي الأرباح (يومي / أسبوعي / شهري)
- سجل العمليات المالية التفصيلي
- طلب سحب الأرباح
- حالة طلبات السحب السابقة (معلقة / موافق عليها / مرفوضة)

---

### 📊 الإحصائيات (Stats)
- عدد الطلبات المنجزة
- التقييم الوسطي من العملاء
- معدل القبول والإنجاز
- الأرباح عبر الفترات الزمنية المختلفة

---

### 👤 الملف الشخصي
- تعديل البيانات الشخصية
- بيانات المركبة
- الاطلاع على نسبة العمولة المُطبَّقة

## الخدمات المشتركة

### 🔄 WebSockets (الوقت الفعلي)
> يُستخدم لإبلاغ جميع الأطراف فور حدوث أي تغيير

| الحدث | من | إلى |
|-------|-----|-----|
| طلب جديد | العميل | المدير + السائق |
| قبول الطلب | السائق | العميل |
| تحديث الحالة | السائق | العميل + المدير |
| تحديث الموقع | السائق | العميل (الخريطة) |
| إشعار جديد | المدير | أي طرف |

---

### 🔐 نظام المصادقة
- **العملاء**: رقم هاتف + كلمة مرور / Google / Apple
- **السائقون**: رقم هاتف + كلمة مرور (نظام منفصل)
- **المدراء**: بريد إلكتروني + كلمة مرور (مسار `/admin-login`)
- الجلسات محفوظة في قاعدة البيانات (connect-pg-simple)

---

### 📱 دعم التطبيق الجوال (Flutter)
- نقطة API مخصصة لإعدادات التطبيق الجوال (`/api/flutter/app-config`)
- ملف APK جاهز: `app-release-unsigned-aligned.apk`
- إعدادات TWA (Trusted Web Activity) للنشر على متجر Google Play

---

### 🌐 الـ API العامة (Public API)
| Endpoint | الوظيفة |
|----------|---------|
| `GET /api/categories` | جلب جميع التصنيفات |
| `GET /api/restaurants` | جلب المطاعم مع الفلترة |
| `GET /api/special-offers` | جلب العروض النشطة |
| `POST /api/coupons/validate` | التحقق من كوبون خصم |
| `GET /api/settings` | إعدادات التطبيق العامة |
| `GET /api/app-status` | حالة المتجر (مفتوح/مغلق) |

---

### 📍 نظام تتبع الموقع
- السائق يُرسل موقعه الجغرافي دورياً أثناء التوصيل
- العميل يرى الموقع على الخريطة مباشرة
- المدير يرى مواقع جميع السائقين على خريطة واحدة
- الخريطة: Leaflet (react-leaflet)
